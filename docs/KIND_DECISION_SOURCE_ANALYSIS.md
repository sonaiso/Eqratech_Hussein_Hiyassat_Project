# مصدر قرار التصنيف (kind) — تحليل

## الهدف

معرفة **لماذا** حصلت كلمة على تصنيف معيّن (مثل «أداة») في الـ pipeline: هل من كاتالوج الأدوات، المبنيات، الضمائر، إلخ.

## السكريبت

```bash
# تتبع كلمات محددة
PYTHONPATH=src python3 scripts/trace_kind_decision_source.py "مُلْكُ" "وَكِتَٰبٌ" "كَوْكَبٌ" "كَسَبَتْ"

# تتبع عينة من unique_words_no_root_expected ذات kind أداة
PYTHONPATH=src python3 scripts/trace_kind_decision_source.py --csv i3rab/unique_words_no_root_expected.csv --filter-kind أداة --limit 80 --output i3rab/trace_decision_source_adaa_sample.csv
```

النتيجة: CSV بأعمدة `word`, `bare`, `kind`, `kind_ar`, `decision_source`, `source_detail`, `operator_matched`, `operator_prefixes`, `catalog_path`.

## معنى decision_source

| المصدر | المعنى |
|--------|--------|
| `operators_catalog_direct` | الشكل العاري للكلمة = أداتاً في `data/operators_catalog_*.csv` |
| `operators_catalog_peeled` | بعد نزع بادئة (و/ف/ب/ك/ل/س) الباقي يطابق أداة |
| `operators_catalog_compound` | الباقي = أداة1 + أداة2 (مثلاً إنما = إن + ما) |
| `special_catalog` | اسم علم / إشارة / حرف من special_words |
| `prep_clitic` | حرف جر + ضمير متصل (مثل عليه، بهم) |
| `pronoun_detached` | ضمير منفصل |
| `verb_heuristic` | فعل بالوزن/اللواحق/البادئة |
| `noun_default_or_marker` | اسم (افتراضي) |
| `unknown` | غير محدد |

## أمثلة خاطئة (مُصنّفة أداة وهي ليست أداة)

### 1. وَكِتَٰبٌ → أداة (خطأ)

- **السبب:** `_strip_diacritics` يزيل U+0670 (ألف خنجرية) فـ «كتاب» تصبح «كتب».
- **الباقي بعد النزع:** و + ك + ت → الباقي «ب» يطابق أداة «ب» في الكاتالوج، أو compound «ت» + «ب».
- **المصدر المسجّل:** `operators_catalog_compound` أو `operators_catalog_peeled` مع `remainder='ب'` و `matched_operator_bare='ب'`.
- **الإصلاح المقترح:** عدم اعتبار الباقي أداةً عندما يكون حرفاً واحداً فقط (ب، ت، ك، ل، …) إذا طول الكلمة العاري > 2؛ أو تطبيع ٰ → ا قبل/بعد إزالة التشكيل لاستبقاء «كتاب».

### 2. كَوْكَبٌ → أداة (خطأ)

- **السبب:** نزع البوادئ و، ف، ب، ك، ل، س: من «كوكب» نزع ك ثم و ثم ك → الباقي «ب» يطابق أداة «ب».
- **المصدر:** `operators_catalog_peeled`، `prefixes='كوك'`، `remainder='ب'`، `matched_operator_bare='ب'`.
- **الإصلاح المقترح:** منع مطابقة peeled عندما يكون الباقي حرفاً واحداً، أو اشتراط حد أدنى لطول الباقي (مثلاً ≥ 2).

### 3. كَسَبَتْ → أداة (خطأ)

- **السبب:** «كسبت» (فعل ماضٍ مؤنث) تُنزع ك ثم س ثم ب → الباقي «ت» يطابق أداة «ت» (تاء القسم).
- **المصدر:** `operators_catalog_peeled`، `prefixes='كسب'`، `remainder='ت'`، `matched_operator_bare='ت'`.
- **الإصلاح المقترح:** نفس ما سبق: عدم الاعتماد على peeled عندما الباقي حرف واحد، أو إعطاء أولوية لتصنيف الفعل (مثلاً من الوزن أو اللواحق) قبل تطبيق كاتالوج الأدوات.

### 4. كُلِّ → أداة (خطأ)

- **السبب:** «كل» (اسم = كلّ) تُنزع «ك» → الباقي «ل» يطابق أداة «ل».
- **المصدر:** `operators_catalog_peeled`، `prefixes='ك'`، `remainder='ل'`، `matched_operator_bare='ل'`.
- **الإصلاح المقترح:** منع peeled عندما يكون الباقي حرفاً واحداً، أو إضافة «كل» إلى قائمة استثناءات (اسم وليس أداة).

### 5. مُلْكُ

- في التتبع الحالي: مُلْكُ → **اسم علم** من `special_catalog` (excluded_name)، وليس أداة. إذا ظهرت كـ «أداة» في CSV آخر فقد يكون السياق أو مسار معالجة مختلف.

## خلاصة الإصلاحات المقترحة

1. **منع peeled عندما الباقي حرف واحد:** في `OperatorsCatalog.classify()` (أو في `_peel_prefixes` / بعدها): إذا `len(remainder) == 1` فلا تعتبر النتيجة مطابقة أداة (أو اشترط أن يكون الباقي من قائمة أدوات متعددة الأحرف فقط).
2. **تطبيع ألف خنجرية (U+0670):** في `_strip_diacritics` أو قبل استدعائها: استبدال ٰ بـ ا حتى لا تتحول «كتاب» إلى «كتب».
3. **مراعاة طول الكلمة:** عدم تطبيق compound أو peeled على كلمات طويلة (مثلاً ≥ 4 أحرف) عندما يكون «الباقي» حرفاً واحداً فقط.

بعد تطبيق هذه التعديلات يُفضّل إعادة تشغيل السكريبت على العينة والتحقق من انخفاض التصنيف الخاطئ لأسماء مثل كوكب، كتاب، كسبت، كل.
